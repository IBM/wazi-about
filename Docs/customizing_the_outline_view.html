# Customizing the Outline view

The Outline view shows a symbol tree of the currently active editor. IBM Z Open Editor maps each symbol to a language element in the COBOL, PL/I, or HLASM file.

However, you might not want every symbol to display in the Outline view. To control which ones appear, you can modify the User or Workspace settings.

# Enabling or disabling Outline view symbols

  1. Navigate to your User and Workspace settings using the following VS Code menu command:

    • Windows/Linux: File > Preferences > Settings
    • MacOS: Code > Preferences > Settings

    Note: VS Code provides different scopes for defining these settings. For more information, see User preferences versus workspace preferences.

  2. Type Outline in the search box to filter the settings.

  3. Enable or disable the outline property settings based on your needs. You need to refer to Outline view symbol mapping reference to map the language elements with the VS Code outline view setting properties.

    Note that each setting's name does not necessarily represent the semantic meaning of its Outline view element. For example, the programming language HLASM does not contain any classes. However, the IBM Z Open Editor uses the outline.showClasses symbol ( ) to represent any HLASM label that is branched-to in the file. This symbol was chosen because it visually reminds us of branching. Therefore, refer to the table below to know which Outline view symbols map to which Visual Studio Code settings.

To modify Visual Studio Code Settings in JSON form, open the settings file from the Command Palette. It is not recommended to change the Default settings.

Command Palette (⇧⌘P) (Ctrl+Shift+P) Settings
> Open Settings (JSON) User
> Open Default Settings (JSON) Default
> Open Workspace Settings (JSON) Workspace

# Outline view symbol mapping reference

Note that some symbols appear multiple times for different settings. This is expected behavior. For more information on this, refer to the Product Icon Reference documentation searching for identifiers with symbol- as a prefix.

Note that each Visual Studio Code Setting shown below is in JSON form. In the Settings Editor, it will be shown in UI form. This table highlights the relationship.
