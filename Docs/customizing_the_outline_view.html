# Customizing the Outline View

The Outline view shows a symbol tree of the currently active editor. The IBM Z Open Editor maps each symbol to a language element in the COBOL, PL/I, or HLASM file.

However, you may not want every symbol to display in the Outline view. To enable/disable which ones appear, you can modify the User or Workspace Settings.

# Enabling / Disabling Symbols

Navigate to your User and Workspace settings using the following VS Code menu command:

  • Windows/Linux: File > Preferences > Settings
  • MacOS: Code > Preferences > Settings

Type Outline in the search box to filter the settings.

VS Code provides different scopes for defining these settings. See here for me details.

# Important Notice

Please note that each setting's name does not necessarily represent the semantic meaning of its Outline view element. For example, the programming language HLASM does not contain any classes. However, the IBM Z Open Editor uses the outline.showClasses symbol ( ) to represent any HLASM label which is branched-to in the file. This symbol was chosen because it visually reminds us of branching. Therefore, refer to the table below to know which Outline view symbols map to which Visual Studio Code settings.

# Symbol Mappings

Note that some symbols appear multiple times for different settings. This is expected behavior. For more information on this, refer to the Product Icon Reference documentation searching for identifiers with symbol- as a prefix.
