# Setting file associations

Within the zopeneditor.datasets property, you can specify data set names that should contain COBOL, PL/I, or HLASM programs, copybooks, includes, and JCL. This means that when you open members of these data sets using Zowe Explorer in the editor, they shall be considered the designated type of file. In the following example, when you open any member of ZOWE.SAMPLE.COBOL in MVS™, the content of the member is opened in a COBOL editor, and likewise for PL/I, HLASM, and JCL. To accomplish such a mapping in the IDE, save the property into your settings file, and the IDE uses the entry to map the file to the appropriate editor.

Rule 1: By default, members in the COBOL data set that end with .COBOL, .COBCOPY, or .COPYBOOK are supposed to be COBOL, COPYBOOKs, or COBCOPY. Members in the PL/I data set that end with .PLI, .PLINC, or .INCLUDE are supposed to be PL/I, PLINC, or INCLUDES. Members in the HLASM data set that end with .ASM are supposed to be HLASM. If your team or company uses COPYBOOK data sets for HLASM copymembers, you can move those settings from cobolDatasets to the hlasmDatasets parameter.

Rule 2: The data sets are listed in a mapping files.associations in the preferences.

"zopeneditor.datasets": {
    "cobolDatasets": [
      ".COBOL",
      ".COB",
      ".COBCOPY",
      ".COPYBOOK",
      ".COPY"
    ],
    "jclDatasets": [
      ".JCL"
    ],
    "pl1Datasets": [
      ".PL1",
      ".PLI",
      ".INC",
      ".INCLUDE"
    ],
    "hlasmDatasets": [".ASM"]
  },

Alternatively, you can directly specify an array of associations instead of using the "zopeneditor.dataset" property above. For example,

"files.associations": {
  "*.COBOL*": "cobol",
  "*.COB*": "cobol",
  "*.COBCOPY*": "cobol",
  "*.COPYBOOK*": "cobol",
  "*.COPY*": "cobol",
  "*.PL1*": "pl1",
  "*.PLI*": "pl1",
  "*.INC*": "pl1",
  "*.INCLUDE*": "pl1",
  "*.JCL*": "jcl",
  "*.ASM*": "hlasm"
}

Again, if you prefer COPYBOOK data sets be opened as hlasm data sets, you can designate that in this setting.

Such a setting is used to recognize any file name, including data set and member names, which contain the noted strings as the appropriate type of file.

Using this setting, you can navigate to the path specified, and see a downloaded copy of all remote files edited via the Zowe VS Code extension.

Note: All file temps are deleted on startup and shutdown.
